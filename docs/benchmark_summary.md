# 压测汇总

本页基于 [benchmark_proof.md](../benchmark_proof.md) 中的原始 `wrk` 输出整理。

## 测试条件
- 压测命令：`wrk -t8 -c400 -d30s --latency http://127.0.0.1:9006/picture.html`
- v1.0 启动命令：`./server -p 9006 -l 0 -m 3 -o 0 -s 8 -t 8 -c 1 -a 0`
- v2.0 启动命令：`./server -p 9006 -l 0 -m 3 -o 0 -s 8 -t 8 -c 1`
- v1.0 模式：`proactor`
- v2.0 模式：主从 reactor

## 三次结果
| 版本 | Run 1 QPS | Run 2 QPS | Run 3 QPS | 平均 QPS | 平均延迟 | P99 延迟 |
|---|---:|---:|---:|---:|---:|---:|
| v1.0 | 12401.92 | 12296.35 | 12212.73 | 12303.67 | 32.44ms | 42.61ms |
| v2.0 | 7429.96 | 7481.87 | 7492.30 | 7468.04 | 53.27ms | 61.63ms |

## 对比结论
- v1.0 平均 QPS 比 v2.0 高 `4835.63`
- 按 v2.0 作为基准，v1.0 约高 `64.7%`
- 本次条件下，v2.0 的平均延迟更高，尾延迟也更差（是不是很奇怪？为什么优化后的版本居然会不如原版呢？别着急，关于这个的答案我放倒了architecture大海的尽头，你去找吧！）
